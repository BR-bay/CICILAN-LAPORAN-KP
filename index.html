BAB I

PENDAHULUAN

1.1 Deskripsi Instansi

Dinas Pendidikan Kota Pekanbaru merupakan instansi pemerintah yang bertanggung jawab dalam pengelolaan dan pengembangan pendidikan di tingkat kota. Instansi ini membawahi seluruh sekolah dasar negeri dan swasta yang berjumlah sekitar 300 sekolah di wilayah Kota Pekanbaru. Tugas utama Dinas Pendidikan adalah merumuskan kebijakan, menyelenggarakan, mengoordinasikan, dan mengendalikan kegiatan di bidang pendidikan.

Selama melaksanakan magang, penulis ditempatkan di Bidang Pembinaan Sekolah Dasar yang khusus menangani pengelolaan data dan arsip sekolah dasar. Permasalahan utama yang dihadapi adalah masih manualnya pengelolaan arsip data siswa, guru, dan administrasi sekolah yang menyebabkan lambatnya proses pengambilan keputusan dan pelaporan.

1.2 Visi, Misi dan Tujuan Instansi

Visi Dinas Pendidikan Kota Pekanbaru adalah "Terwujudnya Pendidikan yang Berkualitas, Berkarakter, dan Berdaya Saing untuk Mewujudkan Pekanbaru sebagai Kota Madani".

Misi instansi ini adalah:

  1. Meningkatkan kualitas dan aksesibilitas pendidikan bagi seluruh masyarakat
  2. Mengembangkan pendidikan karakter yang berbasis kearifan lokal
  3. Meningkatkan profesionalisme pendidik dan tenaga kependidikan
  4. Mengembangkan sistem manajemen pendidikan yang akuntabel dan transparan
  5. Memperkuat sarana dan prasarana pendidikan yang memadai dan merata

Tujuan dari magang ini adalah mengembangkan sistem arsip data digital yang terintegrasi untuk seluruh sekolah dasar di Kota Pekanbaru guna mendukung misi keempat Dinas Pendidikan yaitu mengembangkan sistem manajemen pendidikan yang akuntabel dan transparan.

1.3 Struktur Organisasi

Struktur organisasi Dinas Pendidikan Kota Pekanbaru terdiri dari Kepala Dinas, Sekretaris, dan empat bidang yaitu Bidang Pembinaan SD, Bidang Pembinaan SMP, Bidang Pembinaan PAUD dan PNF, serta Bidang PTK. Selain itu terdapat unit pelaksana teknis dan staf administrasi.

Gambar 1.1. Struktur Organisasi Dinas Pendidikan Kota Pekanbaru

[Gambar Struktur Organisasi]

Penulis selama magang berada di bawah pembinaan langsung Kepala Seksi Data dan Informasi pada Bidang Pembinaan Sekolah Dasar. Struktur ini memungkinkan koordinasi yang efektif dengan sekolah-sekolah dasar dalam implementasi sistem arsip data.

1.4 Proses Bisnis dalam Instansi

Proses bisnis utama di Dinas Pendidikan Kota Pekanbaru terkait dengan pengelolaan data pendidikan meliputi:

  1. Penerimaan dan verifikasi data siswa dari sekolah
  2. Pengolahan dan analisis data pendidikan
  3. Pelaporan data ke pemerintah provinsi dan pusat
  4. Distribusi informasi ke sekolah-sekolah
  5. Monitoring dan evaluasi program pendidikan

Selama ini, proses bisnis tersebut masih dilakukan secara manual dengan menggunakan aplikasi office dan pengiriman data melalui email atau media fisik. Hal ini menyebabkan lambatnya proses, rentannya kesalahan data, dan kesulitan dalam melacak arsip historis.

Dengan adanya sistem arsip data digital yang dikembangkan, diharapkan seluruh proses bisnis dapat berjalan lebih efisien, akurat, dan transparan. Sistem ini akan mengintegrasikan seluruh sekolah dasar dalam satu platform terpadu untuk pengelolaan arsip data.

2.2 Hasil Pekerjaan Secara Umum

Hasil utama dari magang ini adalah terciptanya sebuah sistem arsip data digital untuk sekolah dasar di Kota Pekanbaru. Sistem ini dibangun menggunakan PHP Native, MySQL, CSS murni, dan JavaScript murni dengan arsitektur yang terstruktur.

Gambar 2.1. Diagram Use Case Sistem Arsip Data

[Diagram Use Case Sistem]

Fitur-fitur utama yang berhasil diimplementasikan dalam sistem ini antara lain:

  1. Autentikasi multi-level (Admin Kota, Admin Sekolah)
  2. Manajemen data sekolah dan admin sekolah
  3. Manajemen data siswa dan kelas
  4. Upload dan manajemen ijazah digital
  5. Pencarian data real-time
  6. Backup data otomatis
  7. Log aktivitas pengguna
  8. Antarmuka responsif dengan dark mode

Sistem telah diuji dengan data contoh dari 10 sekolah dasar dan berhasil menangani lebih dari 5.000 data siswa serta 1.200 arsip ijazah digital. Waktu respons sistem untuk pencarian data berada di bawah 2 detik bahkan dengan data yang besar.

BAB IV

PENUTUP

4.1 Kesimpulan

Berdasarkan pelaksanaan magang yang telah dilakukan selama 45 hari kerja di Dinas Pendidikan Kota Pekanbaru, dapat disimpulkan bahwa:

  1. Kebutuhan akan sistem arsip data digital untuk sekolah dasar di Kota Pekanbaru sangat mendesak mengingat banyaknya kendala dalam pengelolaan data secara manual.
  2. Sistem arsip data yang dikembangkan telah berhasil memenuhi kebutuhan dasar dengan fitur-fitur seperti manajemen data sekolah, siswa, ijazah digital, serta pencarian data yang efisien.
  3. Penerapan teknologi PHP Native, MySQL, CSS murni, dan JavaScript murni terbukti cukup untuk membangun sistem yang robust dan scalable untuk menangani data dari 300 sekolah dasar.
  4. Pengujian sistem menunjukkan performa yang memuaskan dengan waktu respons cepat dan kemampuan menangani data dalam jumlah besar.
  5. Magang ini memberikan pengalaman berharga dalam penerapan ilmu yang diperoleh di perkuliahan ke dalam dunia kerja nyata, sekaligus mengembangkan soft skills yang diperlukan dalam lingkungan profesional.

Dengan demikian, sistem arsip data digital ini diharapkan dapat menjadi solusi bagi permasalahan pengelolaan data di Dinas Pendidikan Kota Pekanbaru dan meningkatkan efisiensi serta akurasi dalam pengelolaan arsip pendidikan.

4.2 Saran

Berdasarkan pengalaman selama melaksanakan magang dan mengembangkan sistem arsip data, berikut adalah saran untuk pengembangan lebih lanjut:

  1. Perlu dilakukan pelatihan secara berkala untuk admin sekolah agar dapat memanfaatkan sistem secara maksimal dan memahami prosedur backup data.
  2. Pengembangan fitur tambahan seperti integrasi dengan sistem Dapodik untuk sinkronisasi data otomatis akan sangat meningkatkan efisiensi.
  3. Penambahan modul analitik dan dashboard untuk visualisasi data akan membantu dalam pengambilan keputusan berdasarkan data.
  4. Perlu dilakukan optimasi database secara berkala terutama ketika jumlah data sudah sangat besar untuk mempertahankan performa sistem.
  5. Implementasi mekanisme disaster recovery dan redundancy untuk memastikan ketersediaan sistem secara terus-menerus.
  6. Untuk mahasiswa yang akan melaksanakan magang berikutnya, disarankan untuk mempelajari framework PHP seperti Laravel atau CodeIgniter untuk pengembangan yang lebih cepat dan terstruktur.
  7. Perlu adanya MoU antara Universitas dengan Dinas Pendidikan untuk kelanjutan pengembangan sistem dan magang mahasiswa berikutnya.

Dengan implementasi saran-saran di atas, sistem arsip data digital ini dapat terus dikembangkan menjadi lebih komprehensif dan bermanfaat bagi dunia pendidikan di Kota Pekanbaru.

Berikut adalah contoh kode program dari sistem arsip data yang dikembangkan:

Konfigurasi Database (config/database.php)

<?php
define('DB_HOST', 'localhost');
define('DB_NAME', 'sistem_arsip_data');
define('DB_USER', 'root');
define('DB_PASS', '');

class Database {
    private $host = DB_HOST;
    private $db_name = DB_NAME;
    private $username = DB_USER;
    private $password = DB_PASS;
    public $conn;

    public function getConnection() {
        $this->conn = null;
        try {
            $this->conn = new PDO("mysql:host=" . $this->host . ";dbname=" . $this->db_name, 
                                $this->username, $this->password);
            $this->conn->exec("set names utf8");
        } catch(PDOException $exception) {
            echo "Connection error: " . $exception->getMessage();
        }
        return $this->conn;
    }
}
?>

Model Sekolah (app/models/SekolahModel.php)

<?php
class SekolahModel {
    private $conn;
    private $table_name = "sekolah";

    public function __construct($db) {
        $this->conn = $db;
    }

    public function readAll() {
        $query = "SELECT * FROM " . $this->table_name . " ORDER BY nama_sekolah ASC";
        $stmt = $this->conn->prepare($query);
        $stmt->execute();
        return $stmt;
    }

    public function create($data) {
        $query = "INSERT INTO " . $this->table_name . " 
                 SET npsn=:npsn, nama_sekolah=:nama_sekolah, alamat=:alamat, 
                     jenis=:jenis, status=:status, email=:email, telepon=:telepon";
        
        $stmt = $this->conn->prepare($query);
        
        // Bind parameters
        $stmt->bindParam(":npsn", $data['npsn']);
        $stmt->bindParam(":nama_sekolah", $data['nama_sekolah']);
        $stmt->bindParam(":alamat", $data['alamat']);
        $stmt->bindParam(":jenis", $data['jenis']);
        $stmt->bindParam(":status", $data['status']);
        $stmt->bindParam(":email", $data['email']);
        $stmt->bindParam(":telepon", $data['telepon']);
        
        if($stmt->execute()) {
            return true;
        }
        return false;
    }
}
?>
